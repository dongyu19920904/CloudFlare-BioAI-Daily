# AI日报写作优化分析 - 基于 baoyu.io 写作建议

## 📚 文章核心观点总结

根据 [baoyu.io 写作建议](https://baoyu.io/translations/writing-advice) 文章，核心写作原则包括：

### 1. **宏观心法：降低读者理解成本，提升阅读愉悦感**
- 确保读者在理解内容时所需的脑力消耗最小化
- 同时提升阅读的愉悦感
- 明确目标读者，使用他们熟悉的语言

### 2. **句子长度：变换句式，增强节奏感**
- 避免连续使用相同长度的句子
- 通过变换句子长度提高文笔质量
- 保持读者的注意力

### 3. **遣词造句：避免重复，提升精确度**
- 避免连续使用相同的词语或结构
- 避免重复使用相同的词汇或句式
- 尝试使用同义词或不同的表达方式

### 4. **主动语态：增强叙述生动性**
- 选择使用主动语态可以使叙述更加生动
- 突出动作的执行者
- 吸引读者的注意力

### 5. **具体描述：增强读者代入感**
- 具体的描述可以让读者更容易沉浸在内容中
- 提供具体的数据、案例或实例
- 而非笼统的概述

### 6. **展示而非陈述：引导读者思考**
- 将"陈述"转化为"展示"
- 通过具体的例子或场景来展示观点
- 而非直接陈述结论

### 7. **关注读者情感：保持互动性**
- 关注读者的情感变化
- 保持互动性，提高读者的参与度
- 可以通过提问、引导思考等方式

## ⚠️ 可以优化的方向

### 1. **句子长度变化（未明确要求）**

**当前状态：**
- ❌ Prompt 中只要求"用短句"，但没有强调"变换句子长度"
- ❌ 没有要求避免连续使用相同长度的句子
- ❌ 实际输出中可能出现连续多个相似长度的句子

**文章建议：**
> 变换句子的长度可以提高文笔质量，保持读者的注意力。避免连续使用相同长度的句子，适当穿插短句和长句，创造出有节奏感的文本。

**优化建议：**
- 在 prompt 中明确要求：**"交替使用短句（5-10字）和长句（15-25字），避免连续3句以上使用相同长度"**
- 添加示例展示句子长度变化的节奏感

**示例对比：**
```
❌ 当前（句子长度单调）：
OpenAI 发布 GPT-5 Turbo。推理速度提升 3 倍。API 价格下调 50%。开发者狂喜。

✅ 优化后（有节奏感）：
OpenAI 发布 GPT-5 Turbo，推理速度提升 3 倍，API 价格下调 50%。开发者狂喜。
就在大家以为 GPT-4 已经无敌的时候，这家公司突然甩出了这个炸弹。
```

---

### 2. **避免重复表达（未明确要求）**

**当前状态：**
- ❌ Prompt 中没有明确要求避免重复使用相同的词语
- ❌ 没有要求避免连续使用相同的句式结构
- ❌ 实际输出中可能出现："XX 发布了... XX 推出了... XX 宣布了..." 这种重复

**文章建议：**
> 避免连续使用相同的词语或结构会使读者感到单调。尝试使用同义词或不同的表达方式。

**优化建议：**
- 在 prompt 中添加：**"避免连续使用相同的动词开头（如连续3句都用'发布'），尝试使用：发布、推出、宣布、上线、开放、开源、推出、亮相等不同表达"**
- 要求检查：**"如果连续3条新闻都用相同句式，必须改写至少1条"**

**示例对比：**
```
❌ 当前（重复"发布"）：
OpenAI 发布了 GPT-5。
Meta 发布了 Llama-4。
Google 发布了 Gemini 3。

✅ 优化后（多样化表达）：
OpenAI 发布 GPT-5。
Meta 开源 Llama-4。
Google 推出 Gemini 3。
```

---

### 3. **主动语态强化（需要更明确）**

**当前状态：**
- ⚠️ 虽然要求避免公文腔，但没有明确强调主动语态
- ⚠️ 没有明确禁止被动语态
- ⚠️ 实际输出中可能出现："该模型被开发"、"功能被推出" 等被动表达

**文章建议：**
> 选择使用主动语态可以使叙述更加生动，吸引读者的注意力。突出动作的执行者。

**优化建议：**
- 在 prompt 中明确要求：**"优先使用主动语态，明确动作的执行者"**
- 添加禁止项：**"❌ 禁止使用被动语态（'被开发'、'被推出'、'被发现'）"**
- 提供转换示例

**示例对比：**
```
❌ 被动语态：
该模型被 OpenAI 开发出来。
新功能被添加到系统中。

✅ 主动语态：
OpenAI 开发了该模型。
OpenAI 为系统添加了新功能。
```

---

### 4. **展示 vs 陈述（需要强化）**

**当前状态：**
- ⚠️ Prompt 中有"拒绝抽象，给我画面"，但未明确区分"展示"和"陈述"
- ⚠️ 实际输出中可能仍有大量陈述性语言

**文章建议：**
> 将"陈述"转化为"展示"可以使内容更具吸引力。通过具体的例子或场景来展示观点，而非直接陈述结论。

**优化建议：**
- 在 prompt 中明确区分：**"展示：通过具体场景、对话、对比来呈现；陈述：直接说结论"**
- 要求：**"每条新闻的开头必须用'展示'而非'陈述'"**

**示例对比：**
```
❌ 陈述：
这个工具提高了工作效率。

✅ 展示：
以前整理这堆文档要熬通宵，现在用这个工具，两分钟就能下班去吃火锅。
```

---

### 5. **句子开头多样化（未涉及）**

**当前状态：**
- ❌ Prompt 中完全没有提到句子开头的问题
- ❌ 实际输出中可能出现连续多个句子都以相同方式开头（如都以"XX 公司"开头）

**文章建议：**
> 避免连续使用相同的词语开头。

**优化建议：**
- 添加要求：**"避免连续3句以上以相同方式开头（如都从公司名开始），尝试：时间开头、场景开头、问题开头、数据开头等"**

**示例对比：**
```
❌ 当前（都以公司名开头）：
OpenAI 发布了 GPT-5。
Meta 开源了 Llama-4。
Google 推出了 Gemini 3。

✅ 优化后（多样化开头）：
就在大家以为 GPT-4 已经无敌的时候，OpenAI 突然发布了 GPT-5。
Meta 也不甘示弱，开源了 Llama-4。
而 Google 的 Gemini 3，更是直接跑分碾压。
```

### 6. **感官细节（未涉及）**

**当前状态：**
- ❌ Prompt 中没有要求使用感官细节（视觉、听觉、触觉等）
- ❌ 主要依赖数据和逻辑，缺乏感官描述

**文章建议：**
> 使用具体的感官细节来展示情节，让读者更容易沉浸。

**优化建议：**
- 添加要求：**"在描述产品/工具时，适当加入感官细节：'界面简洁清爽'、'响应速度快到几乎无延迟'、'生成的图片细节丰富'等"**
- 但要注意：**"不要过度使用，保持专业性和可读性"**

### 7. **读者情感连接（需要强化）**

**当前状态：**
- ⚠️ 有"像朋友一样聊天"，但未明确强调情感连接
- ⚠️ 未要求关注读者的情感变化

**文章建议：**
> 关注读者的情感变化，保持互动性，可以提高读者的参与度。

**优化建议：**
- 添加要求：**"在每条新闻的结尾，考虑读者的情感反应：'开发者狂喜'、'等等党又赢了'、'设计师要失业了'等"**
- 但要注意：**"情感表达要自然，不要刻意煽情"**
